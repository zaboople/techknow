External Transition:
    Behavior: Always executes the exit action of the source state and the entry action of the target state.
    Use Case: Standard movement from State A to State B, or a "self-transition" (A to A) where you want to reset/re-initialize the state.
Internal Transition:
    Behavior: Executes an action without leaving the state. No exit or entry actions are triggered.
    Use Case: Handling an event that needs to run logic (like updating a counter) but shouldn't disrupt the current state's lifecycle.
Local Transition:
    Behavior: Used in hierarchical (nested) states. It doesn't exit the superstate if the transition is between that superstate and one of its own substates.
    Use Case: Moving between parent and child states without triggering the parent's full exit/entry logic.
